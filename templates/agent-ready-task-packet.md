# Agent-Ready Task Packet

## Goal

State the outcome the agent should produce.

## Context

List the specific channels, files, repos, docs, tickets, or URLs the agent may use.

## Allowed Tools

Name the tools the agent may call.

## Boundaries

List anything the agent must not read, modify, publish, send, delete, or expose.

## Approval Gates

Name the steps that require human approval.

## Output

Describe the expected artifact, format, and destination.

## Verification

Define how the agent should prove the work is complete.

## Budget

Set time, cost, tool-call, or retry limits.

## Failure Handling

Explain what the agent should do if blocked.
