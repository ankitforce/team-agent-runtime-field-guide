# Team Agent Runtime Taxonomy

## Surface

The place where the agent appears: Slack, IDE, browser, inbox, ticket queue, dashboard, repo, or document system.

## Context

The information the agent can use for a task: messages, docs, code, tickets, files, databases, logs, or prior decisions.

## Tools

The external capabilities the agent can invoke: search, code edits, browser actions, API calls, data queries, ticket updates, publishing, or messaging.

## Permissions

The policy boundary between read, suggest, modify, approve, publish, send, and delete.

## Memory

The retained state that helps future runs: user preferences, project facts, channel summaries, decisions, or prior task outcomes.

## Budget

The limits that prevent uncontrolled work: token budget, wall-clock time, tool-call count, spend cap, retry count, and queue priority.

## Logs

The evidence trail: prompts, context used, tool calls, outputs, failures, approvals, costs, and final artifacts.

## Verification

The checks that make work trustworthy: tests, citations, diffs, rendered previews, source links, human approval, and rollback evidence.
